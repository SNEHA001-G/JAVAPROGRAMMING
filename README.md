Overview
This project is a console-based application that includes multiple tasks aimed at building various systems for practical learning. Below are the five tasks you have completed:

1. ATM Interface System
Purpose: Created a simple ATM interface system that allows users to interact with their bank account.

Key Features:

BankAccount Class: Stores account balance and allows deposit, withdrawal, and balance checking.

ATM Class: Displays a user-friendly menu for withdrawing money, depositing money, and checking the balance.

Input Validation: Ensures proper input (e.g., valid account number format) and provides feedback for invalid choices.

Transaction Logic: Handles withdrawal and deposit processes while ensuring proper balance management.

2. Quiz Application
Purpose: Developed a quiz application with multiple-choice questions and a timer.

Key Features:

Quiz Questions: Stored a list of questions with multiple-choice options and correct answers.

Timer: Implemented a timer for each question to limit the time a user has to answer.

Question Display: Showed one question at a time with options for the user to select.

Answer Submission: Allowed users to submit answers within the given time.

Score Calculation: Kept track of the score based on correct answers.

Result Screen: Displayed the final score and a summary of correct and incorrect answers.

3. Course Registration System
Purpose: Developed a course registration system where students can register and drop courses.

Key Features:

Course Database: Stored course information such as course code, title, description, and schedule.

Student Registration: Allowed students to register for available courses and view course details.

Course Removal: Enabled students to drop courses they were registered for.

Validation: Ensured that students can only register for courses with available slots.

4. Student Information System
Purpose: Built a system for storing and managing student records and course registration details.

Key Features:

Student Database: Stored student information such as student ID, name, and courses they are registered for.

Course Management: Handled student course registration and removal, as well as course availability updates.

Display of Available Courses: Presented the list of available courses with capacity details and schedules.

Student Interaction: Allowed students to select courses and manage their registration status.

5. Complete Student Course Registration System
Purpose: Integrated the functionality of previous tasks to build a complete student course registration system.

Key Features:

Course and Student Databases: Combined the course and student information databases.

Registration and Course Management: Enabled students to view available courses, register, and drop courses.

Course Capacity: Ensured that courses are only offered if there are available slots for students.

Efficient Interaction: Allowed users to interact with the system in a seamless manner for registering and modifying courses.

Technologies Used
Java: Used Java for object-oriented programming to create classes for managing ATM operations, quiz functionality, and course registration system.
